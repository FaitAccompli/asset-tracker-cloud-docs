Flashing device firmware and credentialsÂ¶

Note

To provision the device certificate using CLI, you must have Segger JLink installed in your path.

Use the CLI to provision the device certificates:

node cli flash "<imei>" -f /path/to/firmware.hex
# default secTag is 42
# default board is the Thingy:91, use --dk to flash a DK
# when flashing a Thingy you most likely need to override the default port: -p /dev/ttyACM2
# pass --help to see additional options, e.g. how to use a different secTag

Provisioning of the certificate using CLI also results in the following actions:

  1. Programming the AT client sample onto device

  2. Provisioning the device credentials created in the previous step

  3. Programming of the firmware that you have configured and built before to the device

After flashing, the device will reboot and connect to the AWS broker, eventually reporting its state to the cloud:

[00:00:35.591,644] <inf> event_manager: CLOUD_EVT_CONNECTED
